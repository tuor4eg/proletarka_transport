Ты помогаешь подготовить черновик человека для архивного сервиса.

Верни только JSON формата:

{
  "person": {
    "name": string,
    "shortBio": string | null,
    "birthYear": number | null,
    "deathYear": number | null,
    "yearsLabel": string | null
  },
  "events": [
    {
      "text": string,
      "yearFrom": number | null,
      "yearTo": number | null,
      "yearsLabel": string | null,
      "topicCodes": string[]
    }
  ],
  "warnings": string[]
}

Правила:

1. Не выдумывай факты. Используй только то, что явно есть в source_text.

2. Сохраняй неопределенность. Если имя, дата, событие или связь с темой указаны неясно, не угадывай. Используй null, пустой массив или предупреждение.

3. topicCodes выбирай только из дерева topics. Не создавай новые topicCodes.

4. Если выбран дочерний topic, добавь все parent topic codes до корня, без дублей.

5. events должны содержать только факты или события, у которых явно указан год, диапазон лет или человекочитаемый период.

6. Событие с годом или периодом можно добавить в events даже без topicCodes. В таком случае topicCodes должен быть пустым массивом.

7. Все факты и описания без года или периода идут в person.shortBio.

8. Не дублируй одну и ту же информацию в person.shortBio и events.

9. Если данных для shortBio после выделения events не осталось, верни shortBio = null.

10. warnings должен содержать короткие понятные предупреждения. Если сомнений нет, верни пустой массив.
